## ✅ Applied (2026-07-18, later session): identify-funnel redesign — type/genus characteristics popups, anatomy-aware color stepper, pattern step, moderator upgrade

Big, user-requested redesign of `/identify`'s guided funnel, planned and built in 5 stages (`web/components/coral-identify-funnel.tsx` + supporting libs). Full plan is in the session transcript; summary below.

1. **"What type is it?" step** (renamed from "shape") — each of the 6 category chips and, once a type is picked, each genus chip get an "ⓘ" info trigger opening a popup (`components/characteristics-popup.tsx`) with hand-authored characteristics text (`lib/coral-characteristics.ts` — 6 categories + all 27 genera, from general reef-hobby knowledge, not cited sources — same provisional posture as the seed hex colors) plus a `PhotoCarousel` (`components/photo-carousel.tsx`, up to 5 slides). **No real photo sourcing was possible** (the live DB has only 8 community photos total) — real hero photos fill slots first (`getCategoryShowcasePhotos`/`getGenusShowcasePhotos` in `lib/wiki.ts`), remaining slots fall back to illustrative `ColorSwatch` art (same deterministic-pattern approach already used elsewhere, never fake/hotlinked photography). All fetched server-side once (`getIdentifyShowcaseData`) and passed as props — no new client-side server actions/loading states.
2. **Anatomy-aware color step-through** replaces the old single flat "which of 10 colors" list. `lib/anatomy-steps.ts`'s `STEP_GROUPS` maps each `anatomy_template_code` to beginner-facing steps: zoanthid/paly → **Mouth, Skirt, Tentacles** (exactly as requested); branching SPS → **Growth tip, Skin and Structure, Tentacles**; similar for the other 5 templates. Two real DB additions from user feedback (`sql/supabase/29_anatomy_template_refinements.sql`, applied live + mirrored into `coral_trait_schema.sql`): `tentacle` added to `branching_sps` (color still contributes even though not usually shown extended), and a new `bubble_tip` element type added to `mushroom_coral` (optional — not every mushroom has one). **Optional steps** (leather's Stalk, mushroom's Bubbles) render a "Not on this coral" skip toggle rather than forcing a pick.
3. **Color-proportion input, deliberately "not overbearing"**: picking one color for a step shows no percent UI at all (implicit 100%); 2 colors shows one slider (`components/color-percent-split.tsx`, 10%-snapped); 3+ shows small steppers with a live total hint. Feeds a new conservative ranking bonus in `scoreCoralMatch` (`lib/color-match.ts` — optional third param, dominant-color match +0.05, pattern match +0.03, unit-tested to never outrank a genuinely stronger coverage match) — the ranking-bonus design anticipated in `docs/color-percent-feature-brief.md` §7.
4. **Pattern-recognition step** — 6 pattern chips (spotted/mottled/banded/tipped/ringed/rainbow), each popup rendering a real `ColorSwatch` example (a real coral from the registry when one documents that pattern via `getPatternExampleCoral`, else a generic hex pair) instead of a photo. Selecting a pattern is additive-only, never a filter.
5. **Moderator side** (`app/moderate/actions.ts`, `components/color-moderation.tsx`): new `getColorEntryModerationList()` — "which corals have I set up, and when did I last touch them" (sorted by most-recent `color_ranges.updated_at`, "never entered" quick filter), row click pre-selects the taxon in the existing editor. The editor now groups rows by the same `STEP_GROUPS` framework (with a soft "adds up to X%" hint per group). Hex tool upgraded: each color stop now has a real `<input type="color">` synced to its hex text (was one comma-separated text field), add/remove-stop buttons, and a live preview using the actual `ColorSwatch` component. Pattern `<select>` gets inline helper text per option (what "Range" vs "Solid" etc. actually means) — confirmed the existing gradient rendering (e.g. Sunset Acropora's orange→pink) was untouched by this change.

**Real bug caught and fixed during live verification**: the type/genus/pattern chip's "ⓘ" info button was initially nested inside the chip's own `<button>` — invalid HTML (button-in-button), causing a real hydration mismatch in the browser console. Fixed by restructuring into `ChipWithInfo` — a `<div class="funnel-chip">` wrapping two sibling `<button>`s (select + info) instead of nesting. ## ✅ Applied (2026-07-14): identify-MVP Phase 1 — guided color-match funnel

**Product context**: a design critique of `/identify` surfaced that none of the site's three existing identification paths actually delivered the tagline ("know your coral by its actual colors — not a guess"): community voting is literally the "old way" the landing page argues against, the anatomy-position color sampler requires expertise a novice doesn't have (where does a "skirt" end?), and reference photos suffer the same white-balance problem the tagline names. Decision: make "what colors do you see" (anatomy-free, order-free, novice-operable) the front door, with anatomy/hex precision demoted to a *confirmation* step and community voting demoted to a *fallback*. 1. **`web/lib/color-match.ts`** (+ `color-match.test.ts`, 30 tests, `npm test` via new `vitest` devDependency) — pure, dependency-free core:
   - `hexToFamily(hex)` bins any hex into one of 10 beginner-recognizable families (red/orange/yellow/green/teal/blue/purple/pink/brown/cream), aligned to the schema's pre-existing `named_colors` table.    - `scoreCoralMatch(userFamilies, coralFamilies)` — coverage-weighted (85%) with precision as a tiebreaker (15%), so a focused 2-color exact match outranks a 6-color coral that merely happens to contain those 2 colors, while a coral missing one of the user's colors still ranks below one with full coverage.
2. **`getCoralsForColorMatch()` / `getFunnelCategories()`** (`web/lib/wiki.ts`) — server-side data fetch that bins every morph's `color_ranges` into families once (via the module above), so the client component never re-derives anatomy/hex data, just scores against pre-binned families.
3. **`CoralIdentifyFunnel`** (`web/components/coral-identify-funnel.tsx`) — the new `/identify` front door: (1) optional shape/category picker reusing the 6 real categories from `24_coral_categories.sql`, (2) tap-to-select color chips, (3) a live-scored, ranked shortlist of real registry corals (hero photo or color-tile fallback, matched-vs-missing color chips, link to the wiki page as the confirmation step).
